# WorldCup Copilot 参赛原型与源代码材料设计

## 目标

从现有 WorldCup Copilot 项目中整理一套可直接用于赛事报名或作品申报的材料，清楚展示产品流程、AIGC 创新点和关键实现。材料必须来自当前真实项目，不虚构尚未实现的功能。

4. `source-code\*.txt`

Word 和 Markdown 中的文字均可直接复制到报名系统。PNG 用于上传“产品原型图”，TXT 用于上传或复制“部分源代码”。

## 原型图范围

原型采用现有产品界面截图，统一裁切、编号和添加简短功能注释，不重画另一套视觉系统。

### 01 赛事内容机会池

- 展示比赛卡片、机会评分、热点雷达和筛选能力。
- 说明系统如何从比赛和热点中发现内容机会。

### 02 单场比赛数据与证据

- 展示比分、基础统计、关键事件和数据来源。
- 说明后续生成内容受比赛事实约束。

### 03 热点关联与推荐选题

- 展示与当前比赛相关的热点短榜。
- 展示客观资讯、专业分析、二创、轻松整活等选题角度。

### 04 多平台内容生成

- 展示平台、生成类型和风格类型选择。
- 突出 B 站主推，并体现微博、小红书、抖音和公众号的差异化输出。

### 05 证据约束风险审稿

- 展示待审稿件、AI 审核结果、具体风险点和改写建议。
- 突出“有证据通过、超出证据范围才提示”的审稿机制。

### 06 Word 方案导出

- 展示可直接发布版、编辑参考版、风险提示和证据来源。
- 说明结果可导出为 Word，形成完整交付闭环。

## 核心源代码范围

只节选能够解释核心创新的代码，不提交整个项目。

1. 热点传播价值评分：`lib/hot/valueScoring.ts`
2. AI 选题与内容生成：`lib/ai/deepseek-workflow.ts`、`lib/ai/platform-draft.ts`
3. 赛事证据构建：`lib/services/evidenceService.ts`
4. 证据约束风险审稿：`lib/ai/review-draft.ts`
5. Word 报告导出：`lib/word-export.ts`

每组代码附件包含：

- 模块用途
- 输入与输出
- 核心代码节选
- 与产品流程的对应关系

## 脱敏规则

- 不复制 `.env.local`、`.env.production` 或任何 API Key。
- 不包含账号、Cookie、部署令牌和第三方服务密钥。
- 不伪造真实新闻或比赛事件。
- 示例数据明确标注为示例；真实数据保留来源名称。
- 删除与核心创新无关的样式、路由和兼容代码，保证节选容易阅读。

## 文档结构

1. 项目名称与一句话定位
